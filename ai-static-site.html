Build a Static Website

What a static site is, what it can do, what it can't, and how to build one for free. This is how YourAgInfo.com was built.

What Is a Static Website?

A static website is a set of files — HTML, CSS, and JavaScript — that sit on a server and get sent to anyone who visits. There's no database, no server-side code running, no login system processing in the background. The page is the same for everyone who visits it.

Farm Analogy Think of a static website like a printed brochure. Once it's printed, it says the same thing to everyone who picks it up. A dynamic website (with a database) is more like a phone call — the answer changes depending on who's calling and what they ask.

YourAgInfo.com is a static site. Every page you see — the seed data, the crop planner, the farm programs page — is just HTML files served directly from GitHub. No server is doing calculations when you visit. The JavaScript runs in your browser to make the pages interactive, but the files themselves are static.

What Can a Static Site Do?

Static sites are more powerful than you might think. But they do have limits.

What It CAN Do

  • Display information, photos, and videos
  • Look completely professional and modern
  • Work on phones, tablets, and desktops
  • Run interactive calculators and tools (like the crop planner)
  • Show data from JavaScript files
  • Include forms that send to email services
  • Load fast — really fast
  • Host for free on GitHub Pages
  • Use a custom domain (yourbusiness.com)
  • Be updated by editing files and pushing to GitHub

What It CAN'T Do

  • Let users create accounts or log in securely
  • Save user data to a database
  • Process payments or orders
  • Send emails from the server
  • Show different content to different users
  • Run background jobs or scheduled tasks
  • Connect to private APIs that need server keys
  • Handle complex multi-step forms with server validation

The key question: Does anyone need to save data or log in? If no, a static site is probably all you need. If yes, you need the full stack (see the Farm Database guide).

Why Static Sites Are Great for Small Businesses

  • Free hosting forever — GitHub Pages hosts your site at no cost. No monthly bills, no surprise charges. The only cost is a custom domain ($10-15/year) if you want one.
  • Blazing fast — Static sites load instantly because there's no database to query, no server to process requests. GitHub serves your files from a global network.
  • Nearly unhackable — No database means no SQL injection. No server means no server vulnerabilities. No login system means no passwords to steal. Static sites are the most secure type of website.
  • Zero maintenance — No server updates, no database backups, no security patches. Once it's up, it just works.
  • Easy to build with AI — Claude can write complete static sites. You describe what you want, Claude writes the HTML/CSS/JS, you push it to GitHub, and it's live.

Good Uses for a Static Site

Business Info Site

Your services, contact info, photos, and hours. Like a digital business card that works 24/7.

Product Showcase

Display your seed lineup, equipment, or services with photos, specs, and pricing.

Data Reference

Yield data, rate charts, planting guides — anything people look up but don't change.

Event or Club Page

Farm bureau chapter, cattlemen's association, 4-H club — schedules, news, and resources.

The Honest Limitations

Static sites are great, but be realistic about what you're getting:

  • Updating content means editing code — There's no admin panel or "Edit" button. To change text or add a page, you edit the HTML file and push it to GitHub. Claude can make changes for you, but it's not like clicking "Edit" in WordPress.
  • No real user interaction — You can't have real login accounts, save customer preferences, or process orders. Basic contact forms work (using services like Formspree), but that's about it.
  • No content management system — If you need to post blog articles, news updates, or product listings frequently, a static site can feel clunky. Each update is a code change.
  • Search isn't built in — If you have hundreds of pages, users can't search through them without adding a third-party service.
  • You need to be comfortable with GitHub — It's not hard, but there's a small learning curve. Claude can help, but you still need to understand the basics of uploading files and committing changes.
When to Upgrade If you find yourself needing logins, databases, or user-submitted data, it's time to move to the full stack. The good news: everything you built as a static site can be upgraded. You don't start over — you add a backend. See the Farm Database guide for what comes next.
